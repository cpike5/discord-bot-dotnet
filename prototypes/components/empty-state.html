Implementation Notes

  • Icon: Use a relevant emoji or illustration that represents the empty state
  • Title: Clear, concise heading that describes what's missing
  • Message: Explain why the content is empty and what the user can do
  • Action: Provide a clear call-to-action button when appropriate
  • Tone: Be helpful and encouraging, not discouraging
  • Context: Tailor the message to the specific situation (first-time vs no results)
  • Variants: Use compact or inline versions for limited spaces
  • Colors: Use colored backgrounds to emphasize state (info, warning, error)

When to Use Empty States

  • First-time user experience (no data created yet)
  • No search or filter results
  • Completed items (inbox zero, all tasks done)
  • Permission or access restrictions
  • Error or failed data loading
  • Temporary states (no notifications, no active users)
