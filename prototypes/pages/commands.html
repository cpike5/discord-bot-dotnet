User Commands

Available to all registered users

/register
User

Request an invite code to register for a web account. The bot will send you a direct message with a unique code that can be used to create your account on the website.

Permission Everyone
Cooldown 5 minutes
Example Usage:
/register
/profile
User

View your account profile information including registration date, account status, and linked Discord information.

Permission Registered Users
Cooldown None
Parameters:
user optional
View another user's public profile information
Example Usage:
/profile /profile user:@JohnDoe
/help
User

Display a list of available commands and helpful information about using the bot. You can also get detailed help for a specific command.

Permission Everyone
Parameters:
command optional
Get detailed information about a specific command
Example Usage:
/help /help command:register
/status
User

Check the current status of your account, including verification status, active sessions, and any pending actions.

Permission Registered Users
Example Usage:
/status

Admin Commands

Restricted to administrators only

/users
Admin

View and manage registered users. Display a list of users with filtering options, or get detailed information about a specific user.

Permission Administrator
Parameters:
action required
Action to perform: list, view, activate, deactivate
user optional
Target user (required for view, activate, deactivate actions)
Example Usage:
/users action:list /users action:view user:@JohnDoe /users action:deactivate user:@Spammer
/invite-codes
Admin

Manage invite codes. View all active, expired, or used codes. Generate bulk codes or revoke existing ones.

Permission Administrator
Parameters:
action required
Action to perform: list, generate, revoke
status optional
Filter codes by status: active, used, expired
count optional
Number of codes to generate (1-10, default: 1)
Example Usage:
/invite-codes action:list /invite-codes action:list status:active /invite-codes action:generate count:5
/roles
Admin

Manage user roles and permissions. Assign or remove roles from users to control their access levels.

Permission Administrator
Parameters:
action required
Action to perform: assign, remove, list
user optional
Target user (required for assign/remove)
role optional
Role name: Admin, Moderator, User
Example Usage:
/roles action:list /roles action:assign user:@JohnDoe role:Moderator /roles action:remove user:@JohnDoe role:Moderator
/audit
Admin

View audit logs of administrative actions, user activities, and system events. Filter by date, user, or action type.

Permission Administrator
Parameters:
days optional
Number of days to look back (default: 7, max: 30)
user optional
Filter logs for a specific user
action optional
Filter by action type
Example Usage:
/audit /audit days:30 /audit user:@JohnDoe days:7

Moderator Commands

Available to moderators and administrators

/warn
Moderator

Issue a warning to a user for policy violations. Warnings are logged and visible to administrators.

Permission Moderator, Administrator
Parameters:
user required
User to warn
reason required
Reason for the warning
Example Usage:
/warn user:@BadUser reason:Spam in chat
/kick
Moderator

Remove a user from the server. The user can rejoin with a new invite link. This action is logged.

Permission Moderator, Administrator
Parameters:
user required
User to kick
reason required
Reason for the kick
Example Usage:
/kick user:@Troublemaker reason:Repeated policy violations
